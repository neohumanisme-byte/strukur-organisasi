## Implemented Features

- Organization tree rendering from `id` and `parent_id`.
- Employee/Admin role behavior.
- Employee read-only JD view and print-to-PDF.
- Admin add/edit position, JD, parent/reporting line, status, document control, incumbents, and movement history.
- Structure validation: duplicate ID, invalid parent, self-parent, circular reporting.
- Search, filter, collapse/expand tree.
- Import/export JSON.
- Reset to default sample data.
- Print-specific JD layout.
- Responsive corporate UI.
- Backend API in `server.js` using Node core modules only.
- SQLite database via built-in `node:sqlite`.
- Auth/session token and RBAC.
- Separate backend tables for positions, job descriptions, document controls, employee assignments, movement history, approvals, versions, sessions, users, and audit logs.
- Backend audit logging.
- Approval workflow records.
- Position version snapshots.
- Frontend backend detection and login/logout UI.
- Backend sync for Admin add/edit/status/import/reset.
- Granular Admin position endpoints:
  - `POST /api/positions`
  - `PUT /api/positions/:id`
  - `PATCH /api/positions/:id/reporting-line`
  - `PATCH /api/positions/:id/status`
- Approval and version governance:
  - `POST /api/positions/:id/submit`
  - `POST /api/positions/:id/draft-revision`
  - `GET /api/positions/:id/versions`
  - `GET /api/positions/:id/versions/compare?from=N&to=M`
  - `POST /api/positions/:id/versions/:version_no/restore`
  - `POST /api/approvals/review`
  - `GET /api/org-data?as_of=YYYY-MM-DD`
- Approved JD lock with draft revision flow for future changes.
- Version comparison and restore-as-draft flow.
- Effective-dated organization view from version snapshots.
- localStorage fallback if opened directly without server.
- Persistent system message panel for backend success/error feedback.
- Loading/disabled state for backend actions.
- Admin `Approvals` and `Audit Log` panels.
- CSV export for Approvals and Audit Log.
- Automated smoke test via `npm test`.

- Version comparison UI is functional but could be polished with stronger visual diff grouping.
- Effective-dated view is read-only and uses version snapshots; no visual timeline control yet.
- No user management UI, password change/reset, or permission management.
- No formal database backup/restore UI.
- No deployment configuration.
